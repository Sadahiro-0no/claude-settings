# handoff — /clear 前の引き継ぎ保存

今のセッションの作業状態を `.claude/handoff.md` に**40行以内**で書き残す。

## 手順

1. `.claude/handoff.md` に次の形式で書く(このセッションの文脈から要約する。ファイル探索はしない):

   ```
   # handoff: <作業の一行要約>(<日付> / <ブランチ名>)
   ## 目的
   <なぜやっているか。1〜2行>
   ## 現在地(いま何が動く / 壊れている / 未検証か)
   <再開者が最初に知るべき状態。例: 「テストは緑。statusline の色だけ未確認」>
   ## 完了済み
   <確定した成果だけ。ファイルパス付き>
   ## 次の一手
   <再開して最初に実行する具体的な1コマンド or 1編集。ここが曖昧だと再開が失敗する>
   ## 再開手順(この順で着手)
   1. <次の一手の続き — コマンド・ファイルパス・関数名・file:line まで具体的に>
   2. <その次>
   ## 重要な決定・注意点
   <再開者が知らないと誤る判断だけ>
   ```

   記入例(この密度を目安に):

   ```
   # handoff: 日次コスト集計の重複計上バグ修正(2026-07-15 / fix/daily-cost)
   ## 目的
   /clear をまたいでも日次合計がズレないようにする。
   ## 現在地(いま何が動く / 壊れている / 未検証か)
   tests/run-tests.sh は緑。実機の statusline 表示は未確認。
   ## 完了済み
   session-budget-guard.sh:160-195 に単調増分の累積ロジックを実装。
   ## 次の一手
   bash tests/run-tests.sh で section 24 が緑のままか確認する。
   ## 再開手順(この順で着手)
   1. bash tests/run-tests.sh(section 24/25 を確認)
   2. 実機で /clear 前後の 📅 表示を目視(statusline.sh:113 付近)
   ## 重要な決定・注意点
   base ファイルはコスト0でも毎回更新する(/clear 直後の取りこぼし防止)。
   ```

2. 引数で補足メモが渡されていれば「重要な決定・注意点」に含める。
3. 書き終えたら1行で報告する:
   「handoff.md を保存しました。/clear 後の新しいセッションに冒頭から自動注入され、1ターン目から再開できます。」

## 判断基準

- **合格基準: このセッションを知らない読み手が、handoff だけを読んで5分以内に手を動かせること。**
  「次の一手」は1つの実行可能なコマンドか編集に絞る(複数並べない・抽象語禁止)。
  「再開手順」は番号付きで、各行にコマンドかファイルパス(可能なら file:line)を必ず含める。
- 「現在地」で"何が未検証か"を必ず明示する。再開者が「完了済み」を過信して壊れた前提で進むのを防ぐ。
- タイトルの一行要約は必須(新セッション冒頭に注入されたとき、何の作業か即断できるように)。
- 会話の経緯や試行錯誤は書かない — 結論と現在地だけ。40行を超えるなら「完了済み」を削って「次の一手」を残す。

## 完了時の後始末(削除 / アーカイブ)

引き継いだ作業が**完了したら handoff.md を片付ける**(陳腐化メモの誤再開を防ぐため)。2通り:

- **既定(削除)**: `.claude/handoff.md` を削除する。
- **アーカイブ運用**: 履歴を残したい/後で「なぜこの設計にしたか」を参照したいときは、削除の代わりに
  `bash ~/.claude/hooks/handoff-archive.sh` を実行する。`.claude/notes/YYYYMMDD-HHMM-<要約>.md`
  に退避される(ファイル名が索引=常駐トークンゼロ)。どちらを既定にするかはユーザーに従う
  (環境変数 `CLAUDE_HANDOFF_ARCHIVE=1` を使っているなら /clear 時の自動スタブも履歴を残す)。

## 並行タスク・過去メモの参照

- **並行タスクを一旦横に置く**: 次に再開する作業だけを `handoff.md`(注入される再開スロット)に置き、
  今は再開しない別タスクは `handoff-archive.sh` で notes/ に退避しておく。
- **過去メモを探す**: `ls -t .claude/notes/`(新しい順=日時+主題の一覧)。キーワードがあれば
  `grep -l "<語>" .claude/notes/*.md` で該当ファイルだけ特定し、その1つだけ読む。全件は読まない。
